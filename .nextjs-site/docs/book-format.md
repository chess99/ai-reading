# 书籍内容规范

本文档说明如何向 `books/` 目录添加新书籍，以及 Markdown 文件的格式要求。

## 文件命名

```
books/<分类>/<作者>-<书名>.md
```

示例：
- `books/商业管理/彼得·德鲁克-卓有成效的管理者.md`
- `books/投资/本杰明·格雷厄姆-聪明的投资者.md`
- `books/商业管理/公司战略/彼得·蒂尔-Zero to One.md`（支持子分类）

## Frontmatter

每个文件必须以 YAML frontmatter 开头：

```yaml
---
slug: <拼音或英文，用连字符，作为 URL 标识>
title: <书名>
author: <作者>
tags: [<标签1>, <标签2>]
---
```

### slug 规则

slug 用于生成书籍的 URL（`/books/<slug>`），要求：

- **中文书名** → 书名拼音，用连字符分隔：`chan-pin-jing-li-shou-ce`
- **英文书名** → 小写加连字符：`zero-to-one`
- **同名书籍** → 加作者前缀区分：`mba-in-a-day-zh` / `mba-in-a-day-en`
- 只用字母、数字和连字符，不含中文或特殊字符

> slug 不填时，系统自动用文件名（含中文）作为后备，URL 会被 percent-encode，能用但不优雅。建议始终填写。

### tags 建议

tags 用于筛选和关联，建议使用已有的标签保持一致性。可通过站点"标签"页查看所有现有标签。

## 内容结构

frontmatter 之后是书籍正文。推荐结构（根据书籍类型取舍）：

```markdown
<一句话概括这本书解决什么问题、给谁看、核心答案是什么>

---

## 核心论点

- [最重要的 3-5 个观点，每条独立成立]

## 章节精华

### [章节名]
- [关键知识点或方法]
- [可操作的步骤/框架]

## 金句摘录

> [值得反复回味的原文句子]

## 行动清单

- [ ] [读完后可以立即去做的具体事项]

## 延伸阅读

- **《书名》（作者）**：[推荐理由]
```

## slug 自动化

项目使用 husky 管理 pre-commit hook，clone 后运行一次即可：

```bash
npm install
```

**hook 行为**：
- commit 时若有 `books/` 下的 `.md` 文件变动，自动检查 slug
- 缺少 slug → 自动生成并写入文件，阻止本次 commit，提示重新 `git add`
- slug 冲突 → 报错并阻止 commit，需手动修改其中一个 slug（如加作者前缀）

也可以手动运行脚本：

```bash
npm run slugify
```

## 分类目录

在 `books/` 下创建子目录即为新分类，站点自动识别，无需任何配置。支持多级子分类。

## 完整示例

```markdown
---
slug: zhuo-you-cheng-xiao-de-guan-li-zhe
title: 卓有成效的管理者
author: 彼得·德鲁克
tags: [管理, 效率, 个人成长]
---

知识工作者要想卓有成效，必须在正确的事情上投入时间，而不是追求忙碌——德鲁克用五个习惯将这一洞见变成可操作的实践。

---

## 核心论点

- 效率是做好该做的事，而非把任何事都做好
- 时间是最稀缺的资源，必须先记录、再整合、再分配
- 贡献导向：始终问"我能贡献什么"，而非"我需要做什么"

## 章节精华

### 时间管理
- 记录时间日志，找出真正的时间去向
- 消除可以委托他人的事、可以不做的事

## 金句摘录

> 卓有成效是可以学会的。

## 行动清单

- [ ] 连续两周记录每天的时间日志
- [ ] 列出当前所有工作，标注哪些可以委托或取消

## 延伸阅读

- **《深度工作》（卡尔·纽波特）**：专注力是卓有成效的前提，与本书形成互补
```
