# 产品开发流程原理

Donald G. Reinertsen 著

## 第一部分：当前正统观念的十二宗罪

当前主流的阶段门（Phase-Gate）流程存在系统性问题：

| 问题 | 描述 | 关键数据 |
|------|------|----------|
| **未能正确量化经济** | 只关注代理变量，不了解与生命周期利润的实际映射 | 仅15%的开发者知道项目的延迟成本 |
| **对队列视而不见** | 意识不到设计在制品的存在 | 仅2%测量队列大小 |
| **崇拜效率** | 忽视高利用率导致的队列成本 | 高管报告平均利用率达98.5% |
| **敌视变异性** | 认为变异性必须消除才能创新 | 65%认为应尽可能消除变异性 |
| **崇拜一致性** | 盲目坚持原计划 | — |
| **大批量制度化** | 阶段门一次性转移100%工作成果 | — |
| **未充分利用节奏** | 异步、大批量交付信息 | — |
| **管理时间线而非队列** | 用详细计划取代队列管理 | — |
| **缺乏WIP约束** | 几乎没有组织使用WIP约束 | 仅3%使用WIP约束 |
| **僵化性** | 追求效率导致灵活性不足 | — |
| **非经济性流动控制** | 缺乏基于经济学的排序方法 | — |
| **过度集中控制** | 中央集权化倾向严重 | — |

**关键发现**：95%的管理者承认在了解所有需求之前就开始了设计，实际上普通开发者只在了解50%需求时就开始设计——这表明正统流程在鼓励不当行为。

## 第三部分：管理队列

### 核心洞见

**真正的问题不是活动缓慢，而是停滞不前。** 资源充足时仍有大量等待时间，答案在于排队现象。

### 关键原则

**Q1-Q4：队列本质**

- **无形库存原理**：产品开发中的库存（DIP）在财务上不可见，只是"磁盘上的比特"
- **排队浪费原理**：队列造成六种经济浪费——周期时间增加、风险增加、变异性增加、成本提高、质量降低（反馈延迟）、心理负面影响
- **排队容量利用率原理**：利用率每增加一点，队列呈指数增长。98.5%的利用率意味着95%的时间在排队
- **高队列状态原理**：大部分损害来自高队列状态（概率低但影响大）

**Q5-Q8：队列动态**

- **变异性线性增加队列**，但影响远小于利用率
- 高利用率运行会放大变异性（自我造成的伤害）
- 共享队列优于独立队列（单队列多服务台降低处理时间方差）
- 上游队列决定到达瓶颈的变异性

**Q9-Q12：队列管理工具**

- **最优队列规模**是经济权衡的结果（U曲线优化）
- 排队规则可降低队列成本，无需减少队列规模
- CFD（累计流量图）可监测需求和产能趋势
- 利特公式：等待时间 = 队列规模 / 处理速率

**Q13-Q16：排队控制**

- 不要控制利用率，要控制队列规模
- 不要控制周期时间，要控制队列规模（周期时间是滞后指标）
- 队列会随机严重失控，并长期保持该状态
- 不能依赖随机性来纠正随机队列

### 九大常见队列位置

| 位置 | 原因 |
|------|------|
| 市场营销 | 产能分散、需求过剩 |
| 分析 | 高专业化、资源稀缺 |
| CAD | 受限产能、按效率测量 |
| 采购 | 为制造优化 |
| 原型制作 | 专业资源、反馈延迟风险大 |
| 测试 | 最常见的关键路径队列 |
| 管理评审 | 顺序阶段门过程 |
| 工装 | 专业资源 |
| 专家 | 稀缺资源、介入晚 |

## 第四部分：利用变异性

### 核心洞见

**变异性不是敌人，而是创造经济价值的工具。** 这与制造业截然不同。产品开发创造的是信息而非物理产品，而创造新信息必须引入不确定性。

### 关键原则

**V1-V4：利用变异性的价值**

- **有益变异性原理**：变异性可以创造经济价值。高风险方案（50%成功概率）期望值可能高于零风险方案
- **非对称收益原理**：收益的非对称性使变异性能够创造价值。药品开发案例：成功$10亿 vs. 失败$10,000
- **最优变异性原理**：变异性既不应最小化也不应最大化。产品开发上行有限（满足需求后无额外收益）、下行无限，因此存在最优变异性
- **最优失败率原理**：50%失败率通常是最优信息生成率

**V5-V11：减少变异性的方法**

- **变异性聚合原理**：组合不相关随机任务时，整体变异性下降
- **短期预测原理**：预测难度随时间跨度呈指数增长，缩短规划周期是最有效的风险降低方法
- **小实验原理**：将大实验分解为多个小实验可降低变异性（系数变异性随√n下降）
- **复用原理**：复用消除完成时间的变异性
- **负协方差原理**：通过反向平衡降低变异性
- **缓冲原理**：缓冲以金钱换取变异性降低，但经济上往往不合算

**V12-V16：降低后果而非减少变异性**

- **变异性后果原理**：降低后果通常比减少变异性本身更好（最被低估的方法）
- **非线性原理**：在系统性能的线性范围内操作，需要"断路器"机制
- **变异性替代原理**：用廉价变异性替代昂贵变异性
- **迭代速度原理**：改善迭代速度通常优于降低缺陷率（迭代速度是指数项）
- **变异性转移原理**：将变异性转移到成本最低的流程阶段

### 关键区分

| 领域 | 收益函数特征 | 正确响应 |
|------|-------------|---------|
| 制造业 | 对称（均值两侧偏离都造成损失） | 最小化变异性 |
| 产品开发 | 非对称（差结果造成损失，好结果创造收益） | 优化变异性；降低后果 |

## 第五部分：缩小批量

### 核心洞见

批量缩小是改善产品开发流程最强大、最经济有效的工具之一。丰田百倍的制造周期改进不是通过找瓶颈、增加产能实现，而是通过缩小批量。

### 关键原则

**B1-B10：建立缩小批量可行性的基础**

- **批量-排队原理**：批量大小与队列大小成正比
- **批量-变异性原理**：大批量固有地增加流动变异性
- **批量-反馈原理**：小批量加速反馈，延迟反馈使返工成本指数增长
- **批量-风险原理**：小批量通过三种方式降低风险（缩短暴露时间、降低单批失败影响、快速反馈）
- **批量-开销原理**：大批量增加而非减少开销
- **批量-效率原理**：大批量的局部效率提升常被系统级损失完全抵消
- **批量-心理学原理**：大批量降低紧迫感和动力
- **批量-滑移原理**：大批量导致指数级成本和进度增长
- **批量-死亡螺旋原理**：大项目产生认知失调
- **批量-最小公倍数原理**：整批被其最限制性元素限制

**B11-B15：批量经济学**

- **经济批量原理**：最优批量是交易成本与持有成本的U型优化（EOQ方程）
- **低交易成本原理**：降低每批交易成本使小批量可行（丰田SMED案例：24小时→10分钟）
- **批量规模不经济原理**：大批量往往产生隐藏的额外不经济
- **批量-打包原理**：小批量允许更精细的产能利用调谐
- **流动性原理**：松耦合使小批量可行

**B16-B22：批量管理策略**

- 最关键的批量是**运输批量**而非生产批量
- 物理接近使小批量通信成为可能
- 优先处理以最低成本增加最大价值的活动序列
- 在攻击瓶颈前先缩小批量
- 批量大小应随经济因素变化动态调整

### 典型问题领域

- 项目范围（过多创新堆积于单项目）
- 项目资金（整体一次性大额资助）
- 项目阶段（阶段间大门造成100%批量转移）
- 需求定义（初期定义100%需求）
- 测试（测试周期过长）
- 图纸发布（全有或全无的发布方式）
- 设计评审（大量图纸集中评审）

## 第六部分：在制品约束

### 核心洞见

WIP约束是控制周期时间的有力工具。与其控制时间（周期时间），不如控制队列大小（WIP）。

### 关键原则

**W1-W5：WIP约束基础**

- 通过约束WIP来控制周期时间和流动
- WIP约束强制相邻过程之间的速率匹配
- 全局约束用于可预测、永久性瓶颈
- 看板系统能更快响应随机瓶颈
- WIP范围允许相邻过程使用不同批量

**W6-W14：响应队列形成**

- 需求阻塞、需求清除、弹性需求（从需求侧控制）
- 资源拉取、兼职资源、重炮资源、T型资源、交叉培训（从供给侧控制）
- 混合改变（通过改变工作组合调节队列）

**W15-W23：WIP约束实践**

- 老龄化原理：观察异常值
- 升级原理：为异常值创建预定的升级流程
- 渐进节流原理：接近队列上限时逐步加强节流
- 差异化服务原理：按工作流区分服务质量
- 自适应WIP约束：随产能变化调整WIP约束
- 扩展控制原理：防止工作无限扩展
- **关键队列原理**：在队列成本最高的地方约束WIP
- **累积减少原理**：小的WIP减少会累积成巨大效果

### 关键论据

> "控制WIP比控制利用率或周期时间更有效。利特定律允许我们直接控制周期时间。"

> "最流行的WIP约束系统是丰田生产系统中的看板系统。"

> "队列管理的真正艺术不在于监测它们和设定限制，而在于达到限制时我们做什么。"

## 第七部分：在不确定性下控制流动

### 核心洞见

仅靠WIP约束不足以完全控制流动系统。需要五类额外工具：拥塞控制、节拍、同步、排序和网络化开发流程。

### 关键概念

**拥塞（Congestion）**

- 高产能利用率与低吞吐量共存的状态
- 吞吐量由交通密度和速度的抛物线关系决定
- 抛物线左侧运营点本质不稳定（再生反馈导致恶性循环）
- 拥塞崩溃：当加载过高时，系统经历吞吐量曲线的突然坍塌

**节拍（Cadence）**

- 通过固定、可预测的节奏将不可预测事件转化为可预测事件
- 核心作用是限制方差累积
- 通过定期重新同步，将状态恢复到控制范围中心

**同步（Synchronization）**

- 让多个事件同时发生
- 当同步相邻过程的批量大小和时间时，可以显著减少队列

**排序（Sequencing）**

- 产品开发中任务持续时间和延迟成本异质，需非FIFO策略
- WSJF（加权最短作业优先）：同时考虑作业长度和延迟成本

**网络化开发流程**

- 将产品开发视为网络而非线性流程
- 借鉴电信网络设计哲学——假设变异性固有，采用对变异性稳健的方法

### 30条原则概要

| 类别 | 核心内容 |
|------|---------|
| **拥塞** | 拥塞崩溃原理；峰值吞吐量原理；可见拥塞原理（用预期流动时间预报）；拥塞定价原理 |
| **节拍** | 定期重新同步；节拍产能裕度；节拍可靠性；节拍会议 |
| **同步** | 同步产能裕度；多项目同步；跨职能同步；谐波原理（嵌套节拍） |
| **排序** | SJF；HDCF；WSJF；本地优先级；轮转原理；抢占原理 |
| **网络** | 定制路由；灵活路由；备选路由；晚绑定；预规划灵活性 |

### 关键论据

> "WIP约束不保证我们选择了高吞吐量运营点，不防止方差累积，也不确保经济最优排序。"

> "异步电路通常更快，但同步电路更稳定。"

> "绑定需求到资源的时机越晚，流动越顺畅。"

## 第八部分：利用快速反馈

### 核心洞见

产品开发的控制目的不是减少变异（这适用于制造业），而是**高效影响整体经济结果**。快速反馈是减少队列、加速学习、稳定控制的关键杠杆。

### 关键原则

**FF1-FF6：经济视角的控制**

- 聚焦对经济成功影响最强的参数
- 寻找"经济影响力"与"控制效率"的交集点
- 选择能预测未来系统行为的控制变量（早干预的成本呈指数下降）
- 设定经济影响相等的触发点
- 产品开发的目标是动态的，不是静态的
- 控制系统应减少负面偏差，也应抓住意外机会

**FF7-FF8：快速反馈的效益**

- 快速反馈减少WIP，形成再生循环（少WIP→短延迟→快反馈→更少WIP）
- 快反馈加速学习，提高信息生成效率

**FF9-FF18：控制系统设计**

- 指标只是控制系统的测量部分
- 短转向半径允许减少规划视野（敏捷性）
- 小批量产生快反馈
- 控制决策背后的经济逻辑，而非参与每个决策
- **本地反馈循环本质上快于全局反馈**（防止方差累积和系统不稳定）
- 为项目建立预定的"变异出口"（泄压阀）
- 将快时间常数回路嵌入慢时间常数回路

**FF19-FF24：人类因素**

- 面对面沟通是最快的反馈形式
- 快反馈让人感知"我的行动导致后果"（产生控制感）
- 大队列难以产生紧迫感
- 人类对大偏差产生去稳反应（面对严重延期常减少而非增加投入）
- 给人们分配他人成果的奖励能对齐行为
- 时间比金钱更稀缺

## 第九部分：实现去中心化控制

### 核心洞见

大多数产品开发组织正朝着中心化方向演进，而这是错误的。真正需要的是针对不确定性进行快速适应的能力——采用"去中心化执行配合中心化协调"的模式。

### 军事原则的借鉴

| 概念 | 传统制造业/瀑布式 | 海军陆战队/流动开发 |
|------|------------------|---------------------|
| **战争本质** | 相对可预测 | 不可预测 |
| **计划角色** | 必须遵守的蓝图 | 保持对齐的基线 |
| **障碍响应** | 必须克服 | 有时绕过更明智 |
| **信息处理** | 向上汇报决策 | 前线授权决策 |
| **失败态度** | 避免犯错 | 不果断比坏决策更危险 |

### 关键原则

**何时去中心化**

- 将控制权去中心化，用于处理"老化快"的问题和稍纵即逝的机会
- 对于不频繁但大型的、或有显著规模经济的问题，应中心化控制
- 根据问题信息的清晰程度采用分层控制

**保持对齐**

- 整体对齐比局部卓越更有价值
- 使命命令（Commander's Intent）：聚焦于"为什么"，而非"做什么"
- 指定主要努力点，将其他活动置于从属地位
- 不仅快速决定改变方向，更要快速执行改变（OODA循环）
- 战术协调应是本地化的（横向沟通）

**技术支撑**

- 决策权去中心化同时必须信息去中心化
- 系统响应速度受限于其频率响应
- 使用内部和外部市场来去中心化控制

**人本维度**

- 主动性是自我强化的
- 快速执行的不完美决策远优于延迟执行的完美决策
- 面对面沟通的带宽和实时性远优于书面替代方案
- 信任通过经验建立

### 关键引用

> "没有战争计划能在战斗第一枪后存活。"（No war plan survives the first shot of battle）

> "An imperfect decision executed rapidly is far better than a perfect decision that is implemented late."

## 贯穿全书的核心主题

### 1. 经济学是唯一真正的基础
全书175条原则都建立在经济框架之上。延迟成本（Cost of Delay）是解锁众多问题的"金钥匙"——它让无形的队列变得可见，让WIP约束变得可量化，让批量经济学变得清晰。

### 2. 制造业与产品开发的根本差异
| 维度 | 制造业 | 产品开发 |
|------|--------|---------|
| 任务性质 | 同质、可重复 | 异质、一次性 |
| 延迟成本 | 同质 | 高度异质 |
| 收益函数 | 对称 | 非对称 |
| 变异性态度 | 消除 | 优化 |
| 批量传递 | 大批量经济 | 小批量必要 |
| 排序策略 | FIFO | WSJF |

### 3. 流动是核心目标
流动意味着工作顺畅地从概念到发布的流动。它通过控制队列、WIP和批次大小来实现，而不是通过增加资源或减少变异。

### 4. 反馈速度是关键杠杆
快反馈形成再生循环：快反馈→小批次→短队列→快反馈。它影响学习速度、控制稳定性、团队心理状态和组织敏捷性。

### 5. 去中心化是应对不确定性的必要能力
不确定性要求快速响应，这只能在本地决策中实现。去中心化不是无纪律，而是通过经济决策规则、使命命令和信任来对齐的敏捷性。

## 实践启示

### 立即可行的行动

1. **量化延迟成本**：如果只做一件事，就量化延迟成本
2. **测量队列**：停止测量利用率，开始测量在制品和队列
3. **减少批量**：缩小每次转移的批量大小
4. **实施WIP约束**：在瓶颈处设置WIP上限
5. **加速反馈**：缩短决策周期和测试反馈时间
6. **去中心化决策**：使用决策规则而非集中审批

### 需要投资的长期能力

- 降低交易成本（使小批量可行）
- 建立流动的物理基础设施
- 培养T型人才
- 构建信任和团队连续性
- 实施经济决策规则系统

## 结论

本书提供了一个全面、连贯的框架，用于理解和管理产品开发中的流动问题。其核心洞见是：传统制造业的精益方法需要根本性重构才能适用于高度不确定性、高度异质性的产品开发环境。

作者的核心主张是：**改善产品开发流是增加利润最有效的方法之一**，而实现这一目标需要同时关注经济学、排队论、组织设计和人类行为。这不是一套规则，而是一套原则——为创造性决策提供信息，而非替代它。
